Assistant Core

The backend tries deterministic commands first, then routes prompts into tool execution, retrieval-backed answers, or plain chat.

The important detail from `api_layer/app.py` is that the command path is not mixed into the retrieval path. The assistant has an explicit fast path before it touches vector search or the LLM.

Execution Flow

These steps are derived directly from the code in `api_layer/app.py`, `core/command/*`, and `core/rag/*`.

1 User input arrives `PromptRequest` payloads are accepted by `/ask_rag`, `/api/prompt`, or `/api/prompt/stream`.
2 Command kernel checks first `run_command_kernel()` calls `evaluate_prompt()` from `core/command/command_kernel.py`.
3A Tool router path Recognized commands become deterministic tool calls handled by `ToolRouter.execute()`.
3B Prompt router path If the kernel returns nothing, `classify_prompt()` decides `system`, `rag`, or `chat`.
4 Optional live-state block `detect_live_topics()` and `SystemProvider.snapshot()` add real runtime context when the prompt mentions services, network, memory, disk, or processes.
5 RAG retrieval The retriever embeds the query, searches Qdrant, filters results, deduplicates chunks, and formats evidence blocks.
6 LLM generation `_ollama_generate()` or `_ollama_stream()` calls the local Ollama runtime.
Tool branch Service execution Deterministic tool routes call service functions directly, for example `service_manager.restart_service()` or `theme_service.apply_theme()`.
7 Transport response The backend returns JSON for non-streaming routes or NDJSON events for the streaming route.

    When Each Path Is Used

    This is the behavior visible in code, not a generalized RAG pattern.

    Deterministic bypass

    Commands like `restart nginx`, `list services`, `firewall status`, and `set theme terminal` are recognized before retrieval. The kernel returns a tool request, and the backend can answer without invoking the LLM at all.

    RAG usage

    Question-shaped prompts and knowledge-oriented prompts are routed into the retrieval flow. The backend chooses a domain, optionally attaches live state, pulls evidence from Qdrant, and then asks the model to answer using that context.

    Fallback usage

    Fallback is the `chat` branch, not a special "RAG failed" branch. Prompts about conversation history or general chat are sent to the chat handler after the kernel finds nothing to execute.

    RAG Pipeline Details

    The supporting files under `core/rag` reveal what gets indexed and how retrieval is constrained.

    Startup and seeding

    • Collection management `init_rag()` calls `ensure_collection()` on startup and only spawns the seed thread when the collection is empty.
    • Seed scope `core/rag/seed.py` explicitly allow-lists project code and a few system config roots, but excludes `static/`, `templates/`, private system files, and large file categories in some contexts.
    • Model warmup A delayed preload thread calls `ensure_model()` about 30 seconds after startup.

    Retrieval behavior

    • Domain-aware search order `retrieve()` prefers `system-help -> docs -> code` for system-help questions, `code -> docs -> system-help` for code questions, and `docs` first otherwise.
    • Weak matches are filtered Results below the score threshold are discarded before prompt construction.
    • Deduplication happens before prompting The retriever removes duplicate file/span matches so repeated chunks do not bloat the final prompt.

    Vector and Embedding Layer

    Chunking `chunker.py` splits files into overlapping text windows and preserves source path, line range, timestamp, and domain metadata.
    ->
    Embeddings `embedder.py` sends each chunk or query to Ollama's `/api/embeddings` endpoint and pads or truncates vectors to the configured dimension.
    ->
    Qdrant storage `vector_store.py` keeps a lazily created Qdrant client, ensures the collection exists, and upserts deterministic point IDs so reseeding is idempotent.
    ->
    Query-time retrieval `retrieve()` asks Qdrant for the top-k similar points, filters them, and packages the evidence for the final model prompt.
